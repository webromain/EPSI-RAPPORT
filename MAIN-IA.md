## Mission Principale : Refonte du Site portdesigns.com

### Contexte et Objectifs

**Objectif stratégique** : Refonte complète du site web pour extension vers particuliers (passage BtoB → BtoC)

**Enjeux** :

- Modernisation de l'image de marque
- Amélioration de la visibilité auprès du grand public
- Accès à nouveau segment marché (particuliers)
- Migration technologique critique (EOL Prestashop 1.7 → 8.2)

**Spécifications techniques** :

- Migration de plateforme : Prestashop 1.7 → Prestashop 8.2
- Hébergement : Serveur mutualisé O2Switch
- Gestion DNS : CloudFlare
- **Deadline V1** : Fin octobre 2025 (environ 2 mois de développement)

**Contraintes du projet** :

- Effectifs limités
- Calendrier serré (développement à partir de septembre après validation maquette)
- Alternance (emploi du temps partagé)
- Stratégie : Site fonctionnel rapid avec fonctionnalités essentielles en priorité

### Phase 1 : Préparation et Cadrage

#### 1.1 Mise en place environnement de développement

En arrivant, le premier défi a été d'acquérir une compréhension approfondie du fonctionnement de Prestashop. Ayant une expérience antérieure de WordPress, il a fallu adapter l'approche car l'architecture et les principes diffèrent significativement.

**Actions menées :**

- Configuration d'un environnement local de test
- Étude approfondie de l'architecture Prestashop 8.2
- Documentation des différences Prestashop 1.7 vs 8.2

#### 1.2 Élaboration du cahier des charges

La création d'un produit digital commence toujours par la définition précise des besoins. Un cahier des charges détaillé a été créé listant :

- Toutes les fonctionnalités requises
- Les modules à installer ou développer
- L'expérience utilisateur souhaitée
- Les aspects design et ergonomie

### Phase 2 : Architecture Modules et Fonctionnalités

PrestaShop fonctionne principalement via un système de modules. La stratégie adoptée a été :

1. **Vérifier** si un module existant résout le besoin
2. **Adapter** un module existant si possible
3. **Développer** un nouveau module seulement si nécessaire et justifié

### Phase 4 : Infrastructure Préproduction

#### 4.1 Diagnostic et stratégie migration BDD

Situation initiale :

- Base de données ancienne (Prestashop 1.7)
- Peu de maintenance antérieure
- Données importantes mais désorganisées (résidus, erreurs, structure incohérente)

**Décision prise** : Repartir de zéro avec Prestashop 8.2 clean + migration sélective des données essentielles

- Avantage : Grand ménage des données
- Faisabilité : Tables remises au bon endroit sans erreurs critiques
- Résultat : Toutes données essentielles conservées, base propre

#### 4.2 Mise en place préproduction

**Étapes :**

1. **Sous-domaine sécurisé** : new.portdesigns.com

   - Sécurisation htaccess (authentification login/password)

2. **Environnement Prestashop 8.2** :

   - Installation neuve instance Prestashop 8.2
   - Gestion multi-versions PHP (2 versions pour 2 instances Prestashop)

3. **Migration données** :

   - Transfert données essentielles Prestashop 1.7 → 8.2
   - Validation intégrité données

4. **Installation modules** :
   - Déploiement complète stack modules listée

#### 4.3 Sélection thème

Recherche et sélection d'un thème Prestashop natif ressemblant au maximum à la maquette Figma pour :

- Accélérer développement
- Simplifier intégration
- Réduire temps de développement custom

### Phase 5 : Développement Préproduction

#### 5.1 Création thème enfant

Implémentation d'un thème enfant (child theme) pour :

- Personnaliser tous éléments au-dessus du thème parent
- Préserver thème original et faciliter futures mises à jour
- Éviter écrasement modifications en cas mise à jour thème parent

**Composants développés :**

- Navigation principale
- Slider hero
- Sections catégories
- Footer
- Pages de produits
- Adaptation responsive

Tous les éléments ont été créés, modifiés, supprimés en respectant précisément la maquette Figma.

### Phase 6 : Mise en Production

**Processus de déploiement :**

1. **Sauvegarde complète** : Backup intégrale Prestashop 1.7 + base données
2. **Copie fichiers** : Déploiement Prestashop 8.2 dans répertoire domaine portdesigns.com
3. **Pointage BDD** : Redirection vers nouvelle base Prestashop 8.2
4. **Go-live** : Site en ligne

**Résultat** : Site entièrement fonctionnel en production

### Phase 7 : Infrastructure Développement et Automatisation

#### 7.1 Problématique initiale

Après mise en ligne, besoin critique :

- Pouvoir développer/tester sans impacter production
- Gérer versions sans manipulation manuelle des fichiers
- Versionner code avec historique
- Automatiser déploiement des changements

#### 7.2 Architecture infrastructure

Architecture complète mise en place :

```
┌─────────────┐        ┌────────────────┐        ┌──────────────┐        ┌─────────────────┐
│  FTP-DEV    │───────→│  VSCode + Git  │───────→│ REPO-PORTDEV │───────→│ FTP-PRODUCTION  │
│  (Source)   │        │  (Work local)  │        │   (GitHub)   │        │    (Deploy)     │
└─────────────┘        └────────────────┘        └──────────────┘        └─────────────────┘
```

**Workflow :**

1. Récupération code depuis FTP dev
2. Travail sur branche développement VSCode avec Git
3. Push modifications vers repo GitHub
4. Merge sur branche main → Déclenchement workflow d'automatisation
5. Propagation automatique changements vers FTP production

#### 7.3 Avantages système

- ✅ **Maintenabilité** : Code versé et historisé
- ✅ **Visibilité** : Suivi des changements et commits
- ✅ **Continuité** : Pas d'interruption développement
- ✅ **Sécurité** : Backups, versioning, rollback possible
- ✅ **Automatisation** : Déploiement sans manipulation manuelle

#### 8.3 Override : an_homeslider - Support Vidéo et Image Mobile

**Module original** : an_homeslider (Slider de bannières accueil)

**Problématiques initialement** :

- Absence support format vidéo (MP4, WebM)
- Pas d'optimisation mobile (images desktop en mob)
- Pas d'adaptation responsive des bannières

**Solutions implémentées** :

**1. Extension base de données**

```sql
ALTER TABLE `ps_an_homeslider_slides_lang`
ADD `image_mobile` varchar(255) NOT NULL DEFAULT "" AFTER `image`;
```

- Ajout colonne image mobile pour chaque diapositive
- Permet stockage image carrée adaptée petit écran

**2. Overrides classes**

- **[anHomeSlides.php](DOCUMENTATION/overrides/an_homeslider/classes/anHomeSlidesOverride.php)** : Gestion modèle données étendues
- **[AdminAnhomeSlidesController.php](DOCUMENTATION/overrides/an_homeslider/controllers/admin/AdminAnhomeSlidesOverride.php)** : Interface admin enrichie
- Support format vidéo dans contrôleur admin
- Gestion image mobile dans interface backoffice

**3. Template responsive**

- **widget.tpl** : Template frontend smart
- Détection type device
- Switch automatique desktop/mobile
- Support lecture vidéo MP4/WebM
- Fallback image si pas vidéo

**4. Processus installation**

- Migration SQL (ajout colonne)
- Copie fichiers override vers `/override/modules/an_homeslider/`
- Vérification charge override au démarrage module
- Vidage cache via backoffice

**5. Maintenance futures mises à jour**
Ajout charge conditionnelle override au début fichiers an_homeslider :

```php
// PORT: Chargement de l'override si existant
$overrideFile = _PS_ROOT_DIR_.'/override/modules/an_homeslider/classes/anHomeSlidesOverride.php';
if (file_exists($overrideFile)) {
    require_once $overrideFile;
}

// Ne déclare la classe que si elle n'existe pas encore
if (!class_exists('anHomeSlides')) {
    // ... code original
}
```

**Bénéfices** :

- ✅ Support vidéo marketing enrichissant
- ✅ Images optimisées par device (performance)
- ✅ Responsive vrai (pas image redimensionnée mobiles)
- ✅ Upgrades module non bloqués

**Emplacement** : `/override/modules/an_homeslider/` + `themes/child_galaxy/modules/an_homeslider/`
